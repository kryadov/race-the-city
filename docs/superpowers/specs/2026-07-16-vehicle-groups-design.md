# Десять новых машин + группировка выбора — дизайн

**Дата:** 2026-07-16
**Версия:** v0.58.0 (одним релизом)
**Статус:** на вычитку

## 1. Задача

Добавить 10 типов транспорта к девяти существующим и сгруппировать выбор машины
в меню, потому что плоский список из 19 штук нечитаем.

Новые типы: авто-кран, минивэн, гусеничный вездеход, аэро-мобиль без колёс,
электромобиль, ретро, цистерна, мотоблок, каток, комбайн.

Было решено делать одним релизом: промежуточное состояние (группы на девяти
машинах или плоский список из 19) само по себе никому не нужно.

## 2. Ключевые решения

| Тема | Решение | Почему |
|------|---------|--------|
| Группировка | По назначению, 4 группы | Понятно с первого взгляда; альтернативы (по характеру езды, по ходовой) дают неровные группы |
| Состав групп | Данными в `vehicles.ts`, не разметкой | Меню не знает про конкретные машины; добавить тип = одна строка |
| Сворачивание | Переиспользуем `section()` из v0.57.0 | Готово, уже помнит состояние в `rtc.menuOpen` |
| Аэромобиль | Парит ~1 м над рельефом | Даёт «без колёс» почти даром; настоящий полёт требует вертикальной физики и камеры — отдельная большая работа |
| Модели | Разделить `model.ts` на `models/` по семьям | 357 строк на 9 моделей → ~700 на 19; файл станет делать слишком много |

## 3. Группы

```
VEHICLE_GROUPS = [
  { key: 'vehGroup.cars',    types: [car, sports, racecar, cabrio, retro, ev, minivan] }
  { key: 'vehGroup.trucks',  types: [truck, lorry, tanker, bus] }
  { key: 'vehGroup.special', types: [crane, roller, combine, tractor, tiller] }
  { key: 'vehGroup.exotic',  types: [motorbike, tracked, hover] }
]
```

Ключи новых типов: `crane`, `minivan`, `tracked`, `hover`, `ev`, `retro`,
`tanker`, `tiller` (мотоблок), `roller`, `combine`.

## 4. Изменения по файлам

**`src/vehicle/vehicles.ts`**
- 10 значений в `VehicleType` и `VEHICLE_TYPES`.
- 10 записей `VehicleSpec`. Характеры через существующие параметры:
  - `crane` — тяжёлый, высокий, вялый руль; близко к `lorry`.
  - `minivan` — как `car`, но медленнее разгон и больше радиус.
  - `tracked` — медленный, огромное `gripLateral` (гусеницы не скользят), тугой поворот.
  - `hover` — средняя скорость, низкое `gripLateral` (парит → скользит), мгновенный руль.
  - `ev` — резкий `accel`, средний `maxSpeed`, хорошее сцепление.
  - `retro` — мягкий: слабый тормоз, малое сцепление, средняя скорость.
  - `tanker` — самый тяжёлый после `lorry`, скользит.
  - `tiller` — крошечный `radius` (~0.5), очень медленный, вертлявый.
  - `roller` — почти не едет, максимальный радиус.
  - `combine` — медленный, большой, тугой.
- Новая карта `HOVERS: Partial<Record<VehicleType, boolean>> = { hover: true }`
  рядом с существующей `LEANS`.
- Новая `VEHICLE_GROUPS` (см. §3).

**`src/vehicle/models/`** (новая директория)
- `cars.ts`, `trucks.ts`, `special.ts`, `exotic.ts` — билдеры по семьям.
- `model.ts` остаётся тонкой склейкой: `BUILDERS`, `STOP_STYLE`, публичный
  `buildVehicleMesh(type)` без изменений сигнатуры — остальной код не трогаем.

**`src/vehicle/car.ts`**
- `stepCar`: если `HOVERS[spec.key]`, то `y = provider.heightAt(x, z) + HOVER_H`,
  иначе как сейчас. `HOVER_H = 1.0` м. Единственная правка физики.

**`src/app/scene.ts`**
- `syncCamera`: у парящего не наклонять кузов по нормали рельефа — висит ровно.

**`src/ui/settingsMenu.ts`**
- `section(key)` → `section(key, parent = panel)`, чтобы группы вкладывались.
- Выбор машины рисуется циклом по `VEHICLE_GROUPS`, а не плоским списком.

**`src/i18n/i18n.ts`**
- 10 подписей машин + 4 названия групп на каждый язык.

## 5. Тестирование

Юнит-тесты на чистых данных (без рендера):
- У каждого типа из `VEHICLE_TYPES` есть: спека в `VEHICLES`, билдер в `BUILDERS`,
  подпись во **всех** языках.
- `VEHICLE_GROUPS` покрывают все типы ровно по одному разу — без пропусков и дублей.
  Это ловит забытый тип при следующем добавлении.
- Парение: у `hover` Y строго выше рельефа; у обычной машины Y равен рельефу.
- Колёса: у `hover` в меше нет колёс (иначе парящая машина с колёсами выглядит глупо).

Внешний вид моделей проверяем глазами — это нормально для игры.

## 6. Что НЕ входит

- Настоящий полёт (высота, перелёт домов) — отдельная работа.
- Проходимость по воде для аэромобиля — вода остаётся препятствием.
- Гусеничная анимация, стрела крана как отдельный механизм, вращение барабана
  катка — модели статичны, кроме уже существующего вращения колёс.
- Кастомные цвета машин — отдельный пункт бэклога.
