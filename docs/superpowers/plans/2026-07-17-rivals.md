# Races with rivals — implementation plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** three AI rivals race the player around the time-trial gate course, with the player's
position shown in the HUD.

**Architecture:** A* over the existing road graph gives a rival a route to the next gate; each
rival is a real `CarState` driven through `stepCar` by the same steer/throttle/brake logic the
autopilot uses. Racing rides on the time trial's gates rather than laying out a second course.

**Tech Stack:** TypeScript, Three.js, vitest (node environment — no DOM in tests unless the test
builds it).

## Global Constraints

- Work on `master`. No feature branches.
- Every new module gets a test file under `test/` mirroring its path.
- Tests run in node, no jsdom: modules touching `document` are tested only through pure exports.
- InstancedMesh is not used here; rival meshes come from `buildVehicleMesh(type)`.
- Comments explain constraints and why, never what the next line does.
- Both languages: every new UI string needs an `en` and a `ru` entry in `src/i18n/i18n.ts`.
- No new dependencies.

/** A graph from a list of positions and undirected links. */
function graphOf(pts: [number, number][], links: [number, number][]): RoadGraph {
  const nodes = pts.map(([x, z]) => ({ x, z, links: [] as number[] }))
  for (const [a, b] of links) {
    nodes[a].links.push(b)
    nodes[b].links.push(a)
  }
  return { nodes, nearest: () => -1 }
}

describe('findRoute', () => {
  it('walks a chain end to end', () => {

      [

        [10, 0],
        [20, 0],
      ],
      [

        [1, 2],
      ],
    )

  it('takes the shorter of two ways round', () => {

    const g = graphOf(

        [10, 0],

        [20, 0],
      ],

      if (i > 0) links.push([i - 1, i])

    const g = graphOf([[0, 0]], [])

```ts
import type { RoadGraph } from './roadGraph'

/**
 * How many nodes a search may settle before it gives up.
 *
 * The road graph is not one connected piece — a river, a motorway or a gap in
 * the OSM data splits it into islands, and a rival stranded on the wrong one
 * would otherwise search every node in the city, every time, on the frame it
 * asks for a route.
 */
const MAX_VISITS = 4000

/** Straight-line distance, in metres — the A* heuristic, and never an overestimate. */
function dist(a: { x: number; z: number }, b: { x: number; z: number }): number {
  return Math.hypot(a.x - b.x, a.z - b.z)
}

/**
 * The shortest way from one node to another, as node indices, `start` and `goal`
 * included.
 *
 * Empty when there is no way there, or when the search hit `maxVisits` first —
 * a caller gets "no route" either way and must have something to do about it.
 */
export function findRoute(
  graph: RoadGraph,
  start: number,
  goal: number,
  maxVisits = MAX_VISITS,
): number[] {
  const nodes = graph.nodes
  if (!nodes[start] || !nodes[goal]) return []
  if (start === goal) return [start]

  const best = new Map<number, number>([[start, 0]])
  const cameFrom = new Map<number, number>()
  const done = new Set<number>()
  // A plain list scanned for the cheapest entry: a heap is faster in principle,
  // but the frontier here is small and this runs a few times a lap, not a frame.
  const open: number[] = [start]
  const priority = new Map<number, number>([[start, dist(nodes[start], nodes[goal])]])

  while (open.length && done.size < maxVisits) {
    let pick = 0
    for (let i = 1; i < open.length; i++) {
      if ((priority.get(open[i]) ?? Infinity) < (priority.get(open[pick]) ?? Infinity)) pick = i
    }
    const at = open.splice(pick, 1)[0]
    if (at === goal) {
      const path = [at]
      for (let n = at; cameFrom.has(n); ) {
        n = cameFrom.get(n) as number
        path.push(n)
      }
      return path.reverse()
    }
    done.add(at)

    const soFar = best.get(at) as number
    for (const next of nodes[at].links) {
      if (done.has(next) || !nodes[next]) continue
      const cost = soFar + dist(nodes[at], nodes[next])
      if (cost >= (best.get(next) ?? Infinity)) continue
      best.set(next, cost)
      cameFrom.set(next, at)
      priority.set(next, cost + dist(nodes[next], nodes[goal]))
      if (!open.includes(next)) open.push(next)
    }
  }
  return []
}
```

Write `src/app/rivals.ts` with exactly this public surface:

```ts
export interface RaceState {
  /** The player's position, 1-based. */
  place: number
  /** How many are racing, the player included. */
  of: number
}

/** What ranking needs to know about a racer. */
export interface Runner {
  taken: number
  x: number
  z: number
}

export function placeOf(you: Runner, rivals: Runner[], gate: Vec2 | null): number
export interface Rivals {
  enabled(): boolean
  setEnabled(on: boolean): void
  reset(roads: Road[], provider: ElevationProvider, car: CarState, course: Vec2[]): void
  update(dt: number, car: CarState, playerTaken: number, course: Vec2[]): RaceState
  dispose(): void
}
export function createRivals(scene: THREE.Scene, grid: SpatialGrid, provider: ElevationProvider): Rivals
```

Requirements:

- `placeOf` — one plus the number of rivals ahead. A rival is ahead when it has taken more gates,
  or has taken the same number and is nearer `gate`. With `gate === null` only the gate count
  ranks. Pure: no module state.
- Three rivals, from `const FIELD: readonly { type: VehicleType; cap: number }[]`, each a
  different vehicle so they read apart at a glance, `cap` being its top speed in m/s. Use caps
  between 20 and 32 — quick enough to be a race, beatable by a player who drives well.
- `reset` builds the road graph once, spawns each rival on the road nearest a point 6 metres to
  the side of the player (offset `(i - 1) * 6` along the player's right, which is
  `(-sin(heading), cos(heading))`), sets its heading toward the first gate, zeroes `taken`, and
  routes it at gate 0. Rebuild the meshes only on the first reset — reuse them after, and
  `dispose` is what frees them.
- `update` — for each rival: walk its route (a node is reached within 6m; skip up to 12 in a
  frame, as junctions are clusters of vertices), steer at the next node with the same
  `angleDelta(car.heading, want) * 1.6` clamped to ±1 that `autopilot.ts` uses, throttle to its
  cap, and `stepCar` it. Within 9m of its gate the rival takes it, wraps `taken`/gate index round
  the course and re-routes. When `findRoute` gives nothing — the gate is on an island the rival
  can't reach — it must not stall: fall back to steering straight at the gate. Returns
  `placeOf(player, rivals, course[playerTaken] ?? null)` and the field size.
- Meshes follow their rival's position, `y` from the state, `rotation.y = -heading + Math.PI / 2`
  (match how `main.ts` orients the player's car — check it and copy it).
- `setEnabled(false)` hides the group; `update` while disabled returns `{ place: 1, of: 1 }`
  without stepping anything.

- [ ] **Step 3: Implement**

- `sparkAt(t, v0, origin)` is ballistics and nothing else: `origin + v0 * t` with
  `- 0.5 * G * t²` on `y`. Export `G` or keep it module-local at about 9 — this is a firework,
  not the car, so it does not want the car's arcade gravity.
- `createFireworks` owns one `THREE.Points` per live burst (a few dozen sparks each, random
  velocities on a sphere, `SIZE = 0.6`, additive blending, `depthWrite: false`, `fog: false`
  — the chase camera looks down through 300-900m of fog and a burst behind it would be grey
  mush). Each spark keeps its own colour, drawn from two or three bright hues per burst.
- A burst fades out over its life (about 2.2s) via the material's `opacity`, then is removed and
  its geometry disposed. `fire` while several are already going must not leak: cap the live
  bursts and drop the oldest.
- `update(dt)` advances every burst's age and rewrites its position attribute from `sparkAt`
  (`needsUpdate = true`).
